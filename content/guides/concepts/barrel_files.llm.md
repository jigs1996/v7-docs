# Barrel Files

> Auto-generated collections of exports for controllers, policies, events, and listeners that consolidate multiple imports into single import statements, reducing import clutter in route files.

## Key Concepts

- **Purpose**: Reduce import clutter by consolidating individual imports
- **Location**: `.adonisjs/server` directory (auto-generated)
- **Optional**: Can use direct imports or disable entirely
- **Auto-sync**: Dev server watches and updates automatically
- **Performance**: Zero impact via lazy loading

## Supported Entity Types

| Barrel File | Import Path | Entity Type |
|-------------|-------------|-------------|
| `controllers.ts` | `#generated/controllers` | Controllers |
| `policies.ts` | `#generated/policies` | Bouncer policies |
| `events.ts` | `#generated/events` | Events |
| `listeners.ts` | `#generated/listeners` | Event listeners |

## Problem: Import Clutter

Without barrel files:
```ts
// start/routes.ts
import router from '@adonisjs/core/services/router'

const NewAccountController = () => import('#controllers/new_account_controller')
const SessionController = () => import('#controllers/session_controller')
const PostsController = () => import('#controllers/posts_controller')
const PostCommentsController = () => import('#controllers/post_comments_controller')

router.get('signup', [NewAccountController, 'create'])
router.get('login', [SessionController, 'create'])
router.get('posts', [PostsController, 'index'])
router.get('posts/:id/comments', [PostCommentsController, 'index'])
```

## Solution: Single Import

With barrel files:
```ts
// start/routes.ts
import router from '@adonisjs/core/services/router'
import { controllers } from '#generated/controllers'

router.get('signup', [controllers.NewAccount, 'create'])
router.get('login', [controllers.Session, 'create'])
router.get('posts', [controllers.Posts, 'index'])
router.get('posts/:id/comments', [controllers.PostComments, 'index'])
```

## Generated Structure

```ts
// .adonisjs/server/controllers.ts
export const controllers = {
  NewAccount: () => import('#controllers/new_account_controller'),
  Session: () => import('#controllers/session_controller'),
  Posts: () => import('#controllers/posts_controller'),
  PostComments: () => import('#controllers/post_comments_controller')
}
```

## Import Aliases

```ts
import { controllers } from '#generated/controllers'
import { events } from '#generated/events'
import { listeners } from '#generated/listeners'
import { policies } from '#generated/policies'
```

- `.adonisjs` directory registered as subpath import in `package.json`
- Uses `#generated` prefix for all barrel files

## Lazy Loading

Controllers wrapped in lazy import functions:
```ts
{
  Posts: () => import('#controllers/posts_controller')
}
```

- Module loaded only when used in route
- Zero performance impact
- On-demand loading identical to direct imports

## Generation

- Auto-generated by `node ace serve`
- Watcher updates on file create/delete
- Do not manually edit `.adonisjs` directory files
- Changes overwritten on regeneration

## Disabling Barrel Files

Configure in `adonisrc.ts`:
```ts
import { indexEntities } from '@adonisjs/core'
import { defineConfig } from '@adonisjs/core/app'

export default defineConfig({
  hooks: {
    init: [
      indexEntities({
        controllers: { enabled: false },
        events: { enabled: false },
        listeners: { enabled: false }
      })
    ]
  }
})
```

After disabling:
- Existing barrel files remain in `.adonisjs/server`
- Manually remove files
- Update code to use direct imports
